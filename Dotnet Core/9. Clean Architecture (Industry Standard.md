# 🚀 Clean Architecture (Industry Standard)

## 🧠 Idea

👉 Separate project into **layers** (independent)

```text
Presentation (API)
    ↓
Application (Business Logic)
    ↓
Domain (Core Rules)
    ↓
Infrastructure (DB, External)
```

---

# 📁 Project Structure (Multi-project)

```text
/MyApp
│
├── MyApp.API              → Controllers
├── MyApp.Application      → Services, DTOs, CQRS
├── MyApp.Domain           → Entities, Interfaces
├── MyApp.Infrastructure   → EF, DB, External services
```

---

# 🧱 Layer Responsibility

## 1️⃣ Domain (Core)

* Entities
* Interfaces
* No dependency on anything

```csharp
public class User
{
    public int Id { get; set; }
    public string Name { get; set; }
}
```

---

## 2️⃣ Application

* Business logic
* DTOs
* CQRS handlers

```csharp
public class UserDto
{
    public string Name { get; set; }
}
```

---

## 3️⃣ Infrastructure

* EF Core
* Repositories

```csharp
public class UserRepository : IUserRepository
{
    private readonly AppDbContext _context;
}
```

---

## 4️⃣ API (Presentation)

* Controllers only

```csharp
[HttpGet]
public async Task<ActionResult<List<UserDto>>> Get()
{
    return Ok(await _service.GetUsers());
}
```

---

# 🧠 Why Clean Architecture?

* Loose coupling ✅
* Easy to test ✅
* Replace DB easily ✅
* Scalable ✅

# 1️⃣ CQRS (Command Query Responsibility Segregation)

## 🧠 Idea

👉 Separate:

```text
READ (Query) ≠ WRITE (Command)
```

---

## ❌ Without CQRS

```csharp
public class UserService
{
    public void CreateUser() { }
    public List<User> GetUsers() { }
}
```

---

## ✅ With CQRS

### Command (Write)

```csharp
public class CreateUserCommand
{
    public string Name { get; set; }
}
```

---

### Query (Read)

```csharp
public class GetUsersQuery { }
```

---

👉 Cleaner + scalable

# 2️⃣ MediatR (🔥 Very Popular)

## 🧠 Idea

👉 Instead of calling service directly:

```csharp
_service.CreateUser()
```

👉 Use mediator:

```csharp
_mediator.Send(command);
```

---

## 📦 Install

```bash
dotnet add package MediatR
```

---

## ✅ Example

### Command

```csharp
public record CreateUserCommand(string Name) : IRequest<int>;
```

---

### Handler

```csharp
public class CreateUserHandler : IRequestHandler<CreateUserCommand, int>
{
    public async Task<int> Handle(CreateUserCommand request, CancellationToken ct)
    {
        // logic
        return 1;
    }
}
```

---

### Controller

```csharp
public async Task<IActionResult> Create(CreateUserCommand cmd)
{
    var result = await _mediator.Send(cmd);
    return Ok(result);
}
```

---

## 🎯 Benefit

* Loose coupling ✅
* Clean architecture friendly ✅
* Easy testing ✅

# 3️⃣ FluentValidation

## 🧠 Problem

❌ Without it:

```csharp
if (string.IsNullOrEmpty(user.Name))
    return BadRequest();
```

---

## 📦 Install

```bash
dotnet add package FluentValidation
```

---

## ✅ Example

```csharp
using FluentValidation;

public class UserValidator : AbstractValidator<UserDto>
{
    public UserValidator()
    {
        RuleFor(x => x.Name).NotEmpty();
    }
}
```

---

## 🔌 Register

```csharp
builder.Services.AddValidatorsFromAssemblyContaining<UserValidator>();
```

---

## 🎯 Benefit

* Clean validation ✅
* Reusable ✅
* No messy if-else ✅

---

# 4️⃣ AutoMapper

## 🧠 Problem

❌ Manual mapping:

```csharp
var dto = new UserDto
{
    Name = user.Name
};
```

---

## 📦 Install

```bash
dotnet add package AutoMapper.Extensions.Microsoft.DependencyInjection
```

---

## ✅ Example

### Profile

```csharp
using AutoMapper;

public class MappingProfile : Profile
{
    public MappingProfile()
    {
        CreateMap<User, UserDto>();
    }
}
```

---

## 🔌 Register

```csharp
builder.Services.AddAutoMapper(typeof(Program));
```

---

## ✅ Use

```csharp
var dto = _mapper.Map<UserDto>(user);
```

---

## 🎯 Benefit

* Clean mapping ✅
* Less code ✅
* Easy maintenance ✅
