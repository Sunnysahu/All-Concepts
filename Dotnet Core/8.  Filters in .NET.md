

# 🚀 Filters in .NET

## 🧠 What are Filters?

👉 Filters are **code that runs before/after controller actions**

```text
Request → Middleware → Filters → Controller → Filters → Response
```

---

## 🔥 Why Filters?

👉 Use when you want to:

* Validate request
* Log action execution
* Modify response
* Handle authorization

---

# 🆚 Middleware vs Filters (Important)

```text
Middleware → runs for ALL requests (global level)

Filters → runs ONLY for MVC actions (controller level)
```

👉 Example:

* Logging all requests → Middleware ✅
* Validate model before controller → Filter ✅

---

# 🎯 Types of Filters

---

# 1️⃣ Action Filter (Most Used)

👉 Runs before & after controller action

---

## ✅ Example

```csharp
using Microsoft.AspNetCore.Mvc.Filters;

public class MyActionFilter : IActionFilter
{
    public void OnActionExecuting(ActionExecutingContext context)
    {
        Console.WriteLine("Before Action");
    }

    public void OnActionExecuted(ActionExecutedContext context)
    {
        Console.WriteLine("After Action");
    }
}
```

---

## 🔌 Apply Filter

### 👉 On Controller

```csharp
[ServiceFilter(typeof(MyActionFilter))]
public class UserController : ControllerBase
{
}
```

---

### 👉 On Method

```csharp
[ServiceFilter(typeof(MyActionFilter))]
[HttpGet]
public IActionResult Get()
{
    return Ok();
}
```

---

# ⚙️ Register in DI

```csharp
builder.Services.AddScoped<MyActionFilter>();
```

---

# 🔥 Real Use Case (Validation)

```csharp
public class ValidateModelFilter : IActionFilter
{
    public void OnActionExecuting(ActionExecutingContext context)
    {
        if (!context.ModelState.IsValid)
        {
            context.Result = new BadRequestObjectResult(context.ModelState);
        }
    }

    public void OnActionExecuted(ActionExecutedContext context) { }
}
```

👉 Stops execution if model is invalid

---

# 2️⃣ Authorization Filter

👉 Runs BEFORE everything

---

## ✅ Example

```csharp
using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.Filters;

public class CustomAuthFilter : IAuthorizationFilter
{
    public void OnAuthorization(AuthorizationFilterContext context)
    {
        var user = context.HttpContext.Request.Headers["user"];

        if (string.IsNullOrEmpty(user))
        {
            context.Result = new UnauthorizedResult();
        }
    }
}
```

---

# 3️⃣ Result Filter

👉 Runs BEFORE/AFTER response is sent

```csharp
public class MyResultFilter : IResultFilter
{
    public void OnResultExecuting(ResultExecutingContext context)
    {
        Console.WriteLine("Before Response");
    }

    public void OnResultExecuted(ResultExecutedContext context)
    {
        Console.WriteLine("After Response");
    }
}
```

---

# 4️⃣ Exception Filter

👉 Handles exceptions at controller level

```csharp
public class MyExceptionFilter : IExceptionFilter
{
    public void OnException(ExceptionContext context)
    {
        context.Result = new ObjectResult("Error occurred")
        {
            StatusCode = 500
        };
    }
}
```

---

# ⚠️ Middleware vs Exception Filter

👉 Use Middleware for:

* Global exception handling ✅

👉 Use Exception Filter for:

* Controller-specific handling

---

# 🔥 Execution Order (Important)

```text
Middleware
 → Authorization Filter
 → Action Filter (Before)
 → Controller Action
 → Action Filter (After)
 → Result Filter
 → Response
```

---

# 🧠 Real Project Usage

| Use Case             | Use                  |
| -------------------- | -------------------- |
| Logging all requests | Middleware           |
| Validate model       | Action Filter        |
| Auth check           | Authorization Filter |
| Global error         | Middleware           |

---

# 💣 Common Mistakes

❌ Putting everything in middleware
❌ Not using filters for validation
❌ Using filters without DI

---

# 🧠 Interview Summary

* Filters = controller-level pipeline

* Types:

  * Action → before/after action
  * Authorization → security
  * Result → response
  * Exception → error

* Middleware vs Filter:

  * Middleware → global
  * Filter → MVC-specific
