# HOC (Higher Order Component)

### 🧠 Theory
- A **Higher Order Component (HOC)** is a **function that takes a component and returns a new component**.  
- Used to **reuse component logic** across multiple components without repeating code.  
- Often used for authentication, theming, logging, or data fetching.  

---

### ✅ Use Case
- Authentication checks  
- Feature toggles  
- Injecting props or behavior into components  

---

### 💻 Example: Basic HOC
```javascript
import React from "react";

function withAuth(Component) {
  return function Wrapped() {
    const isAuth = true; // Replace with real auth logic
    return isAuth ? <Component /> : <p>Login Required</p>;
  };
}

// Usage
function Dashboard() {
  return <h1>Dashboard Content</h1>;
}

export default withAuth(Dashboard);
````

---

### 🔄 How It Works

1. `withAuth` **wraps the original component**
2. Checks `isAuth`
3. Renders either the **original component** or a **fallback UI**
4. You can reuse `withAuth` for multiple components

---

### ⚡ Pro Tip

* HOCs are **pure functions** → don’t modify the original component
* Can chain multiple HOCs for **combined behaviors**

```javascript
export default withAuth(withLogger(Dashboard));
```
