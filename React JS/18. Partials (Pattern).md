# Partials (Pattern)

### 🧠 Theory
- **Partials** are **reusable UI chunks** or small components  
- Not an official React feature, just a **design pattern**  
- Helps **avoid code repetition** and keeps UI consistent  

---

### ✅ Use Case
- Buttons, cards, modals, or form inputs used in multiple places  
- Wrapping repeated UI logic  
- Keeping UI modular  

---

### 💻 Example: Basic Partial Component
```javascript
import React from "react";

// Reusable Button component
function Button({ children, onClick, type = "button" }) {
  return <button type={type} onClick={onClick}>{children}</button>;
}

// Usage
export default function App() {
  return (
    <div>
      <Button onClick={() => alert("Clicked!")}>Click Me</Button>
      <Button onClick={() => console.log("Hello")}>Log Hello</Button>
    </div>
  );
}
````

---

### 🔄 How It Works

1. Create a **small, reusable component** (partial)
2. Pass **props** like `children`, `onClick`, `className`
3. Use it across your app wherever needed

---

### ⚡ Tips

* Partials can **compose other partials**
* Keep them **dumb** (presentational) and **stateless** if possible
* Use for **consistent UI elements** like buttons, inputs, and cards
