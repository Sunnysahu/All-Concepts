## useContext

### 🧠 Theory
- Provides global/shared state without prop drilling  
- Works with a **Provider → Consumer** pattern  
- Any component can access data using `useContext`  

### ✅ Use Case
- Theme (dark/light)  
- Authentication (user data)  
- App-wide settings  

---

## 💻 Advanced Example (Multiple Files)

### 📁 File 1: `AuthContext.js`
```javascript
import { createContext, useState } from "react";

export const AuthContext = createContext();

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);

  const login = (name) => setUser({ name });
  const logout = () => setUser(null);

  return (
    <AuthContext.Provider value={{ user, login, logout }}>
      {children}
    </AuthContext.Provider>
  );
}
```
---
📁 File 2: Navbar.js

```javascript
import { useContext } from "react";
import { AuthContext } from "./AuthContext";

export default function Navbar() {
  const { user, logout } = useContext(AuthContext);

  return (
    <nav>
      <h3>My App</h3>
      {user && (
        <>
          <span>Welcome, {user.name}</span>
          <button onClick={logout}>Logout</button>
        </>
      )}
    </nav>
  );
}

```

---

📁 File 3: Login.js

```javascript
import { useContext } from "react";
import { AuthContext } from "./AuthContext";

export default function Login() {
  const { login } = useContext(AuthContext);

  return (
    <button onClick={() => login("Rohit")}>
      Login
    </button>
  );
}
```

---
📁 File 4: App.js

```javascript
import { AuthProvider } from "./AuthContext";
import Navbar from "./Navbar";
import Login from "./Login";

export default function App() {
  return (
    <AuthProvider>
      <Navbar />
      <Login />
    </AuthProvider>
  );
}

```
---

## 🔄 How Data Flows

### AuthProvider wraps the app → provides global state

### Login → updates state using login()

### Navbar → reads state (user) and reacts to changes

### All components stay in sync automatically

## 🚀 Why This Is Better

### No prop drilling

### Centralized logic (auth in one place)

### Clean separation across files

### Scales easily for real apps
