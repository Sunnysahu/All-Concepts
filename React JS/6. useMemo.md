## useMemo 😄

### 🧠 Theory
- Memoizes expensive calculations  
- Recomputes only when dependencies change  
- Helps avoid unnecessary recalculations and re-renders  

### ✅ Use Case
- Performance optimization  
- Heavy computations inside components  
- Derived state based on props  

---

## 💻 Example: Basic useMemo
```javascript
import { useMemo, useState } from "react";

function computeExpensiveValue(num) {
  console.log("Computing...");
  return num * 2;
}

export default function App() {
  const [count, setCount] = useState(0);
  const [other, setOther] = useState(0);

  const expensiveValue = useMemo(() => computeExpensiveValue(count), [count]);

  return (
    <div>
      <p>Expensive Value: {expensiveValue}</p>
      <button onClick={() => setCount(c => c + 1)}>Increment Count</button>
      <button onClick={() => setOther(o => o + 1)}>Increment Other</button>
    </div>
  );
}
````

---

## 🔄 How It Works

1. `useMemo` stores the result of `computeExpensiveValue(count)`
2. Recomputes **only when `count` changes**
3. Clicking "Increment Other" **does not recompute** → saves performance

---

## ⚡ Tip

* Use for **CPU-heavy calculations**, **not for every small value**
* Combine with `React.memo` for **component-level memoization**

## React.memo

### 🧠 Theory
- `React.memo` is a **higher-order component (HOC)** that memoizes a **whole component**.  
- If the **props haven’t changed**, React **skips re-rendering** that component.  
- Think of it like `useMemo` but for **components instead of values**.  

### ✅ Use Case
- Functional components that **receive props but rarely change**  
- Avoid unnecessary re-renders of **child components**  
- Performance optimization in **large component trees**  

---

### 💻 Example: Basic React.memo
```javascript
import React, { useState } from "react";

const Child = React.memo(({ value }) => {
  console.log("Child rendered");
  return <p>Child Value: {value}</p>;
});

export default function App() {
  const [count, setCount] = useState(0);
  const [other, setOther] = useState(0);

  return (
    <div>
      <Child value={count} />
      <button onClick={() => setCount(c => c + 1)}>Increment Count</button>
      <button onClick={() => setOther(o => o + 1)}>Increment Other</button>
    </div>
  );
}
````

---

### 🔄 Behavior

* Clicking **Increment Count** → `Child` **re-renders** because `value` changed.
* Clicking **Increment Other** → `Child` **does NOT re-render** because props didn’t change.

---

### Component-Level Memoization

* Similar to memoizing a value with `useMemo`, but here we **memoize the component itself**.
* **Purpose:** prevent expensive child components from re-rendering unnecessarily.
* Often combined with `useMemo` or `useCallback` for **props that are functions or objects**, to avoid re-render triggers.

---

### ⚡ Pro Tip

* If props include objects/functions, wrap them in `useMemo` or `useCallback` to **actually benefit from `React.memo`**.
* Example: 

``` javascript
const memoizedCallback = useCallback(() => doSomething(value), [value]);
<Child onClick={memoizedCallback} />
```
