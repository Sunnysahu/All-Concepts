# React Router (2 Ways)

### 🧠 Theory
- Enables **SPA (Single Page Application) navigation**  
- Handles routing **without full page reloads**  

---

### ✅ Method 1: Component-Based Routing
```javascript
import { BrowserRouter, Routes, Route } from "react-router-dom";
import Home from "./Home";
import About from "./About";

export default function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}
````

---

### ✅ Method 2: Data Router (React Router v6.4+)

```javascript
import { createBrowserRouter, RouterProvider } from "react-router-dom";
import Home from "./Home";
import About from "./About";

const router = createBrowserRouter([
  { path: "/", element: <Home /> },
  { path: "/about", element: <About /> }
]);

export default function App() {
  return <RouterProvider router={router} />;
}
```

---

### 🔄 How They Differ

* **Component-based:** classic, simple, defines routes inside JSX
* **Data Router:** v6.4+ feature, supports **loader, action, nested routing**, better for **data fetching and complex apps**
* Both provide **client-side SPA navigation**

# React Router `<Route />` Props Cheat Sheet

A quick reference for all the important props you can use in **React Router v6+**.

---

## **1. `path`**
- **Type:** `string`  
- The URL path for the route.  
- Supports parameters: `/user/:id`  
```javascript
<Route path="/about" element={<About />} />
<Route path="/user/:id" element={<User />} />
````

---

## **2. `element`**

* **Type:** `ReactNode`
* The component or JSX to render when the route matches.

```javascript
<Route path="/" element={<Home />} />
```

---

## **3. `index`**

* **Type:** `boolean`
* Defines an **index route** (default child route for a parent).

```javascript
<Route index element={<DashboardHome />} />
```

---

## **4. `children`**

* **Type:** `<Route />[]`
* Nested routes for **hierarchical routing**.

```javascript
<Route path="/dashboard" element={<Dashboard />}>
  <Route path="stats" element={<Stats />} />
</Route>
```

---

## **5. `caseSensitive`**

* **Type:** `boolean`
* Make path matching **case-sensitive**. Default is `false`.

```javascript
<Route path="/About" element={<About />} caseSensitive />
```

---

## **6. `loader`** (Data Router v6.4+)

* **Type:** `(params) => Promise<any>`
* Function to **load data before rendering** the route.

```javascript
<Route
  path="/users/:id"
  element={<User />}
  loader={({ params }) => fetch(`/api/users/${params.id}`)}
/>
```

---

## **7. `action`** (Data Router v6.4+)

* **Type:** `(params) => Promise<any>`
* Function to **handle form submissions or mutations**.

```javascript
<Route path="/contact" element={<ContactForm />} action={submitForm} />
```

---

## **8. `errorElement`**

* **Type:** `ReactNode`
* Component rendered if **loader/action fails** or route has an error.

```javascript
<Route path="/users/:id" element={<User />} errorElement={<ErrorPage />} />
```

---

## **9. `handle`**

* **Type:** `any`
* Custom object for storing **arbitrary route-specific data**, e.g., for animations or metadata.

```javascript
<Route path="/about" element={<About />} handle={{ animation: "fade" }} />
```

---

## **10. `id`**

* **Type:** `string`
* Optional route ID, useful for **programmatic navigation** or **linking routes** in nested routers.

---

## **11. `lazy`** (React Router v6.8+)

* **Type:** `() => Promise<{ default: React.ComponentType }>`
* Dynamically imports a component for **code splitting**.

```javascript
<Route path="/lazy" lazy={() => import("./LazyComp")} />
```

---

## 🔄 Full Example Combining Props

```javascript
import { createBrowserRouter, RouterProvider, Outlet } from "react-router-dom";

const router = createBrowserRouter([
  {
    path: "/",
    element: <Layout />,
    errorElement: <ErrorPage />,
    children: [
      { index: true, element: <Home /> },
      { path: "about", element: <About /> },
      {
        path: "users/:id",
        element: <User />,
        loader: ({ params }) => fetch(`/api/users/${params.id}`),
        handle: { animation: "fade" }
      }
    ]
  }
]);

export default function App() {
  return <RouterProvider router={router} />;
}
```

---

### ⚡ Tips

* Use **`index` + `children`** for nested layouts
* Use **`loader` + `action` + `errorElement`** for robust data-driven routes
* Combine **`lazy` + `Suspense`** for route-level code splitting
* Keep route definitions **centralized** for easier maintenance
