# createPortal

### 🧠 Theory
- `createPortal` allows you to **render a React component outside the normal DOM hierarchy**  
- Useful for **modals, tooltips, or dropdowns** that need to visually break out of parent containers  

### ✅ Use Case
- Modals  
- Tooltips  
- Floating notifications  
- Dropdowns that escape `overflow: hidden`  

---

### 💻 Example: Basic createPortal
```javascript
import React from "react";
import { createPortal } from "react-dom";

function Modal() {
  return <div className="modal">This is a modal!</div>;
}

export default function App() {
  return createPortal(
    <Modal />,
    document.getElementById("modal-root") // separate div outside main app
  );
}
````

---

### 🔄 How It Works

1. React normally renders components **within their parent DOM node**
2. `createPortal` allows rendering into **any DOM node** (`modal-root`)
3. The component **keeps React state and context** while appearing elsewhere in the DOM

---

### ⚡ Tip

* Keep **modal container divs outside your main app div** in `index.html`

```html
<div id="root"></div>
<div id="modal-root"></div>
```

* Works seamlessly with **animations and state management**
