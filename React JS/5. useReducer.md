## useReducer

### 🧠 Theory
- Handles complex state logic  
- Follows a Redux-like pattern (state + action → new state)  
- Keeps logic predictable and centralized  

### ✅ Use Case
- Complex forms  
- Multiple related state values  
- State machines  

---

## 💻 Basic Example
```javascript
import { useReducer } from "react";

function reducer(state, action) {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 };
    default:
      return state;
  }
}

export default function Counter() {
  const [state, dispatch] = useReducer(reducer, { count: 0 });

  return (
    <>
      <p>{state.count}</p>
      <button onClick={() => dispatch({ type: "increment" })}>
        Increment
      </button>
    </>
  );
}
````

## 💻 Advanced Example (Real-World Form)

### 📁 File 1: `formReducer.js`

```javascript
export const initialState = {
  name: "",
  email: "",
};

export function formReducer(state, action) {
  switch (action.type) {
    case "UPDATE_FIELD":
      return {
        ...state,
        [action.field]: action.value,
      };

    case "RESET":
      return initialState;

    default:
      return state;
  }
}
```

---

### 📁 File 2: `Form.js`

```javascript
import { useReducer } from "react";
import { formReducer, initialState } from "./formReducer";

export default function Form() {
  const [state, dispatch] = useReducer(formReducer, initialState);

  const handleChange = (e) => {
    dispatch({
      type: "UPDATE_FIELD",
      field: e.target.name,
      value: e.target.value,
    });
  };

  return (
    <form>
      <input
        name="name"
        value={state.name}
        onChange={handleChange}
        placeholder="Name"
      />

      <input
        name="email"
        value={state.email}
        onChange={handleChange}
        placeholder="Email"
      />

      <button type="button" onClick={() => dispatch({ type: "RESET" })}>
        Reset
      </button>

      <pre>{JSON.stringify(state, null, 2)}</pre>
    </form>
  );
}
```

## 🔄 How It Works

1. `dispatch(action)` sends an action
2. `reducer(state, action)` decides how state changes
3. Returns a new state → component re-renders

---

## 🚀 Why useReducer over useState?

* Better for **multiple related states**
* Logic is **centralized and reusable**
* Easier to debug and scale

---
## 🔥 useReducer + useContext (Global State Like Redux)

### 🧠 Theory
- `useReducer` → manages complex state  
- `useContext` → shares it globally  
- Together → **Redux-like global store (no external library)**

## 📁 File 1: `Store.js`

```javascript
import { createContext, useReducer } from "react";

export const StoreContext = createContext();

const initialState = {
  count: 0,
};

function reducer(state, action) {
  switch (action.type) {
    case "INCREMENT":
      return { count: state.count + 1 };

    case "DECREMENT":
      return { count: state.count - 1 };

    case "RESET":
      return initialState;

    default:
      return state;
  }
}

export function StoreProvider({ children }) {
  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <StoreContext.Provider value={{ state, dispatch }}>
      {children}
    </StoreContext.Provider>
  );
}
````

## 📁 File 4: `App.js`

```javascript
import { StoreProvider } from "./context/Store";
import Counter from "./components/Counter";
import Controls from "./components/Controls";

export default function App() {
  return (
    <StoreProvider>
      <Counter />
      <Controls />
    </StoreProvider>
  );
}
```

---

## 🔄 Data Flow (Redux Style)

1. `Controls` → dispatches action
2. `reducer` → updates state
3. `StoreProvider` → shares updated state
4. `Counter` → re-renders automatically
