# Error Boundaries

### 🧠 Theory
- React **Error Boundaries** catch **runtime errors in their child component tree**  
- Prevent the entire app from crashing  
- Only works in **class components** (as of now)  

### ✅ Use Case
- Catch rendering errors in components  
- Display fallback UI  
- Log errors for monitoring  

---

### 💻 Example: Basic Error Boundary
```javascript
import React from "react";

class ErrorBoundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError(error) {
    // Update state to display fallback UI
    return { hasError: true };
  }

  componentDidCatch(error, info) {
    // You can log error to an error reporting service
    console.error("Error:", error, info);
  }

  render() {
    return this.state.hasError ? <h1>Something went wrong!</h1> : this.props.children;
  }
}

// Usage
function BuggyComponent() {
  throw new Error("Oops!");
}

export default function App() {
  return (
    <ErrorBoundary>
      <BuggyComponent />
    </ErrorBoundary>
  );
}
````

---

### 🔄 How It Works

1. Child component throws an error
2. `getDerivedStateFromError` updates state → shows **fallback UI**
3. `componentDidCatch` allows **logging the error**
4. App continues running safely

---

### ⚡ Tip

* Wrap **high-risk components** (like data fetching or third-party libraries)
* Combine multiple Error Boundaries for **granular error handling**
